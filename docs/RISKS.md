## 风险应对策略

### 风险规避
- 使用成熟的 ORM（SQLAlchemy）避免 SQL 注入
- 密码使用 bcrypt 哈希存储
- JWT 密钥从环境变量读取，不硬编码

### 风险转移
- 使用云数据库服务（RDS）而非自建
- 使用托管的向量数据库服务（Zilliz Cloud）

### 风险缓解
- 多模型后备策略（OpenAI / DeepSeek / Ollama）
- 本地缓存和断点续传机制
- 定期备份数据库

### 风险接受
- 部分低风险项（如构建时长）可接受在一定范围内
