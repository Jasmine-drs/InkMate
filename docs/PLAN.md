## 一、项目愿景

打造一款**人机协作**的 AI 小说创作工具，以**单元剧小说**为核心场景，强调**章节级生成**与**人工主导**的创作流程。

### 核心理念
- 人机协作、逐章推进
- 人工主导创作，AI 辅助增强
- 单元剧工作流专业化支持

## 三、当前工作重点

### 已完成的主干能力
1. ✅ 第一至第四阶段功能已交付并通过审计复核
2. ✅ 编辑器已支持续写、改写、扩写、版本对比与快捷入口
3. ✅ 设定工坊、大纲 AI、追踪手动提取链路、单项目导出已可用

### 当前收口任务
1. 编辑器高级 AI 操作（缩写/润色/询问/选区一致性检查）
2. 章节批注系统（人工批注 + AI 建议批注）
3. AI 对话上下文与历史联动
4. 章节定稿自动触发追踪提取
5. 批量导出/导出历史工作台与数据导入
6. 恢复 ESLint 并补冒烟测试

### 并行开发建议
1. 编辑器工作台：高级 AI 操作与批注系统共享同一编辑界面，建议由同一负责人串行推进，减少组件冲突。
2. AI 对话链路：上下文拼装、历史加载/清空、前后端消息同步可独立于编辑器工作台并行开发。
3. 追踪与交付链路：章节定稿自动提取、导出工作台、数据导入、ESLint/冒烟测试可作为第三条并行线推进。

## 五、架构摘要

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端      │────▶│   后端      │────▶│   MySQL     │
│ React+TS    │◀────│   FastAPI   │◀────│   (RDBMS)   │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │   Milvus    │
                    │  (向量库)   │
                    └─────────────┘
```

**技术栈**:
- 前端：React 18 + TypeScript + Vite + Ant Design 5.x
- 后端：FastAPI + SQLAlchemy (async) + Python 3.12
- 数据库：MySQL 8.0+ (aiomysql)
- 向量库：Milvus 2.x
- 缓存：Redis
- 认证：JWT (python-jose) + bcrypt
